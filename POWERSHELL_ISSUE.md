# Проблема с PowerShell

## Причина проблемы

PowerShell на Windows имеет проблемы с:
1. **Кодировкой** - особенно с UTF-8 и кириллицей
2. **Путями с кириллицей** - путь `C:\Users\1\Documents\обработка видео` содержит русские буквы
3. **Специальными символами** - пробелы и кириллица в путях могут вызывать ошибки парсинга

## Ошибка которую мы видим

```
ParserError: MissingEndParenthesisInMethodCall
```

Это происходит потому что PowerShell пытается обработать путь с кириллицей и не может правильно распарсить команду.

## Решения

### Решение 1: Использовать CMD вместо PowerShell (рекомендуется)

CMD лучше работает с кириллицей в путях:

```bash
# Откройте CMD (не PowerShell)
cd "C:\Users\1\Documents\обработка видео"
git add -A
git commit -m "Your message"
git push origin main
```

### Решение 2: Использовать bat-файлы

Я создал bat-файлы которые работают через CMD:
- `git-push-now.bat` - для отправки изменений
- `deploy.bat` - для быстрого деплоя

Просто двойной клик по файлу.

### Решение 3: Настроить PowerShell (если нужно использовать PowerShell)

```powershell
# Установить кодировку UTF-8
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
$PSDefaultParameterValues['*:Encoding'] = 'utf8'

# Или использовать chcp
chcp 65001

# Затем выполнить команды
cd "C:\Users\1\Documents\обработка видео"
git add -A
git commit -m "Your message"
git push origin main
```

### Решение 4: Переименовать папку (радикальное решение)

Если проблемы продолжаются, можно переименовать папку проекта на английское имя:
- Было: `обработка видео`
- Стало: `videoeditor` или `video-processing`

Но это потребует изменения пути в настройках.

## Рекомендация

**Используйте CMD или bat-файлы** - это самое простое и надежное решение для работы с Git в проектах с кириллицей в путях.
